# Аудит и переработка на «человечность»

Цель: Помочь автору выявить в тексте следы шаблонности, пустые фразы и однообразный ритм, а также предоставить рекомендации по микрокорректировке на уровне строк/абзацев. Акцент делается на «больше похоже на то, как пишете вы», а не на попытки обойти детекторы.

## Инструкция по применению

```
/authenticity-audit

[Вставьте текст для проверки]
```

## Критерии проверки (с указанием количества и примеров)

Если существует `spec/knowledge/audit-config.json`, используйте его списки связующих слов, пустых фраз и пороговые значения длины предложений; в противном случае используйте встроенные правила по умолчанию.

- Плотность шаблонных связующих слов: количество вхождений и предложения по замене таких слов, как «во-первых, во-вторых, далее, в заключение».
- Общие выражения: например, «в некоторой степени, как всем известно, во многих областях», с предложениями по более конкретной переработке.
- Дисперсия длины предложений: статистика средней длины предложения и дисперсии, маркировка фрагментов с «последовательными длинными» или «последовательными короткими» предложениями и предложения по изменению ритма/объединению.
- Клише: выявление шаблонных фраз, которые можно заменить, с предоставлением двух-трех вариантов переработки, более близких к содержанию.
- Абстрактная перегрузка: указание на абзацы с чрезмерной плотностью абстрактных существительных и предложение по добавлению воспринимаемых деталей.
- Соответствие личности автора: сравнение с «фирменными выражениями/лексикой» из `.specify/memory/personal-voice.md`, маркировка отсутствующих «индивидуальных сигналов» и предложение по их естественному включению.

## Результат переработки

- Пофрагментный вывод: исходное предложение → краткое описание проблемы → вариант с микрокорректировкой → более смелый, персонализированный вариант.
- Контроль степени изменений: приоритет отдается микрокорректировкам, по возможности сохраняется исходный смысл; более масштабные переработки предлагаются только при необходимости.
- Сохранение стиля: стиль и повествовательный угол оригинального текста не меняются, если пользователь явно не запросит иное.
