# Шаблон спецификации вывода рекомендаций ИИ

## Полное приглашение для стороннего ИИ (прямое копирование и использование)

```markdown
# Задача: Анализ романа и предоставление структурированных рекомендаций

## Контекст
Я использую инструмент Novel Writer для создания романа и уже завершил первые [X] глав. Мне нужна ваша помощь в анализе этого контента и предоставлении рекомендаций по улучшению. Ваши предложения будут интегрированы в мои творческие спецификации с помощью автоматизированных инструментов.

## Важные требования
Ваш вывод должен строго соответствовать следующему формату, чтобы программы могли автоматически анализировать и обрабатывать его. Пожалуйста, возвращайте рекомендации непосредственно в формате JSON или Markdown, не добавляя никаких дополнительных пояснений.

## Описание измерений анализа

### 1. Стиль письма (style)
- Последовательность повествовательной точки зрения
- Контроль темпа
- Стиль языка
- Техники описания
- Создание атмосферы

### 2. Формирование персонажей (characters)
- Последовательность характера
- Обоснованность развития
- Четкость мотивации
- Индивидуальность диалогов
- Отношения между персонажами

### 3. Развитие сюжета (plot)
- Логическая связность
- Установка и разрешение предзнаменований
- Напряжение конфликта
- Дизайн кульминации
- Контроль темпа

### 4. Построение мира (worldbuilding)
- Самосогласованность правил
- Богатство деталей
- Культурный фон
- Характеристики эпохи
- Особые системы (например, магия, технологии и т. д.)

### 5. Качество диалогов (dialogue)
- Языковые особенности персонажей
- Эффективность передачи информации
- Роль в продвижении сюжета
- Соответствие фону эпохи
- Выражение эмоций

## Критерии оценки

### 1-3 балла: Серьезные проблемы, требуется значительная переработка
### 4-6 баллов: Проблемы существуют, требуется много изменений
### 7-8 баллов: В целом хорошо, требуется небольшая оптимизация
### 9-10 баллов: Отлично, можно оставить как есть

## Примечания

1. **Конкретность**: Все рекомендации должны указывать на конкретные главы и абзацы.
2. **Практичность**: Рекомендации должны быть напрямую выполнимы автором.
3. **Приоритет**: Четко обозначайте степень срочности для каждого типа рекомендаций.
4. **Сбалансированность**: Указывайте как на проблемы, так и на положительные моменты.
5. **Точность данных**: Номера глав, количество слов и т. д. должны быть точными.

## Обработка часто задаваемых вопросов

### В1: Что делать, если в каком-то измерении нет проблем?
О: Все равно сохраните это поле, но массив `items` может быть пустым, а `priority` установите в "low".

### В2: Что делать, если обнаруженных проблем больше, чем полей в шаблоне?
О: В соответствующей категории вы можете добавить несколько `items`, по одному на каждую проблему.

### В3: Как обрабатывать проблемы, затрагивающие несколько измерений?
О: Запишите проблему в наиболее релевантном измерении, а в `suggestion` укажите на связанные влияния.

### В4: Ограничение по объему?
О: Для формата JSON старайтесь уложиться в 5000 символов, для Markdown — в 3000 слов, чтобы сохранить фокус на главном.

## Интеграция с Novel Writer

Ваш вывод будет интегрирован в систему Novel Writer с помощью следующей команды:
```
/constitution refine [Ваш вывод]
```

Система автоматически:
1. Проанализирует ваши рекомендации.
2. Обновит соответствующие файлы спецификаций.
3. Сгенерирует отчет об улучшениях.
4. Направит дальнейшее творчество.

Поэтому, пожалуйста, убедитесь в точности формата вывода и выполнимости рекомендаций.
