Пользовательский ввод может быть предоставлен напрямую или как аргумент команды — вы **должны** рассмотреть его перед продолжением (если он не пуст).

Пользовательский ввод:
$ARGUMENTS

## Цель

Выявлять и уменьшать двусмысленность или отсутствующие точки принятия решений в плане истории, собирая уточняющую информацию посредством интерактивных вопросов и ответов, а затем записывая результаты в файл истории.

**Примечание**: Этот процесс уточнения должен выполняться и завершаться до этапа `/outline`. Если пользователь явно указывает на пропуск уточнения (например, при исследовательском творчестве), можно продолжить, но необходимо предупредить о повышенном риске доработки на последующих этапах.

## Шаги выполнения

### 1. Начальная проверка

Запустите `{SCRIPT}` для получения текущих путей к истории:
- Разбор JSON для получения `STORY_PATH` и `STORY_NAME`
- Если файл истории не найден, предложите пользователю сначала запустить `/story` для создания плана истории
- Загрузите содержимое файла истории для анализа

### 2. Сканирование структуры на предмет двусмысленности

Проведите полное сканирование плана истории, оценивая ясность каждой категории (Ясно/Частично ясно/Отсутствует):

**Творческая ориентация**
- Целевая аудитория (возрастная группа, гендерная предрасположенность, уровень чтения)
- Позиционирование произведения (коммерческий хит/серьезная литература/жанровая беллетристика)
- Ожидаемый объем (короткий рассказ 3-5万/средний 10-20万/длинный 50万+)

**Мироустройство**
- Точность эпохи (конкретный год/династия/степень вымышленности)
- Правила мира (магическая система/уровень технологий/социальная структура)
- Географический охват (один город/несколько стран/континент/межзвездное пространство)

**Дизайн персонажей**
- Кривая роста главного героя (от неудачника к победителю/гениальный тип/постепенное развитие)
- Основной тон характера главного героя (восторженный/спокойный/коварный/святой)
- Функциональное позиционирование второстепенных персонажей (продвижение сюжета/эмоциональная поддержка/контрастное сравнение)
- Уровень интеллекта антагониста (антагонист с пониженным интеллектом/равный противник/превосходство высшего уровня)

**Повествовательная стратегия**
- Выбор точки зрения (первое лицо/третье лицо с ограничением/всеведущий взгляд)
- Структура временной линии (линейное повествование/обратный ход, вставки/параллельные линии)
- Темп повествования (быстрый темп хита/медленное развитие/сбалансированный)

**Ядро сюжета**
- Тип основного конфликта (человек против человека/человек против природы/человек против общества/человек против себя)
- Четкость основной цели (месть/рост/спасение/исследование)
- Склонность к финалу (счастливый конец/трагедия/открытый финал)

**Стиль и тон**
- Выбор стиля письма (плавный разговорный/классический элегантный/юмористический/суровый реализм)
- Акцент на описании (сцены действия/психологическое описание/атмосфера окружения/продвижение диалога)
- Эмоциональный тон (восторженный и страстный/угнетающий и мрачный/теплый и исцеляющий/душераздирающий)

**Творческие ограничения**
- Обработка чувствительного контента (уровень насилия/масштаб эмоций)
- Ценностная ориентация (позитивная энергия/реализм/критический)
- План обновлений (ежедневные/еженедельные/ежемесячные)

Для каждой категории "частично ясной" или "отсутствующей" генерируются кандидатные вопросы, если:
- Уточнение не оказывает существенного влияния на направление творчества
- Информация лучше определяется на этапе планирования глав

### 3. Генерация очереди приоритетных вопросов

Внутренне генерируется до 5 приоритетных уточняющих вопросов с применением следующих ограничений:
- Общее количество вопросов за всю сессию не более 5
- Каждый вопрос должен быть отвечен одним из следующих способов:
  * Множественный выбор (2-5 взаимоисключающих вариантов)
  * Краткий ответ (ограничение до 5 слов)
- Включать только вопросы, которые существенно влияют на направление творчества
- Обеспечить сбалансированное покрытие категорий, отдавая приоритет областям с высоким влиянием
- Если требуется уточнение более 5 категорий, выбрать 5 с наивысшим показателем (влияние × неопределенность)

### 4. Цикл последовательных вопросов и ответов

**Отображать по одному вопросу за раз**

Формат множественного выбора:
```markdown
| Вариант | Описание |
|------|------|
| A | Подробное описание варианта A |
| B | Подробное описание варианта B |
| C | Подробное описание варианта C |
```

Формат краткого ответа:
```
Пожалуйста, ответьте кратко (до 5 слов): _______
```

**Обработка ответа пользователя**
- Проверка действительности ответа
- При наличии двусмысленности запросить быстрое уточнение
- Записать ответ и перейти к следующему вопросу

**Условия остановки**
- Все ключевые двусмысленности устранены
- Пользователь сигнализирует о завершении ("готово", "достаточно", "нет необходимости")
- Достигнут лимит в 5 вопросов

### 5. Интеграция результатов уточнения

Сразу после каждого принятого ответа:

**При первой интеграции**
- Создать раздел `## Запись уточнений` в плане истории (если он не существует)
- Добавить подзаголовок `### Сессия уточнений [Дата]`

**Формат записи**
```markdown
- Вопрос: [Содержание вопроса] → Ответ: [Ответ пользователя]
```

**Обновление соответствующих разделов**
Обновить соответствующие части плана истории на основе уточняющих данных:
- Творческая ориентация → Обновить обзор истории
- Мироустройство → Обновить мироустройство
- Персонажи → Обновить описание персонажей
- Повествовательная стратегия → Добавить в описание творчества
- Стиль и тон → Добавить в руководство по стилю

### 6. Проверка и сохранение

После каждого обновления проверить:
- Полноту записи уточнений
- Отсутствие оставшихся маркеров двусмысленности, разрешенных новыми ответами
- Отсутствие противоречивых утверждений
- Корректность формата Markdown

Записать обновленное содержимое обратно в файл истории.

### 7. Отчет о завершении

Отчет включает:
- Количество заданных и полученных вопросов
- Путь к обновленному файлу истории
- Список затронутых разделов
- Сводная таблица покрытия:

| Категория | Статус |
|------|------|
| Творческая ориентация | ✅ Уточнено |
| Мироустройство | ✅ Уточнено |
| Дизайн персонажей | ⏸ Отложено до планирования |
| ... | ... |

- Рекомендуемая следующая команда (обычно `/outline`)

## Правила поведения

- Если не обнаружено значимых двусмысленностей, ответить: "Ключевых двусмысленностей, требующих немедленного уточнения, не обнаружено."
- Если файл истории отсутствует, направить пользователя сначала запустить `/story`
- Общий лимит не более 5 вопросов
- Избегать вопросов о чисто технических деталях письма
- Уважать сигналы пользователя о досрочном завершении
- Если полное покрытие достигнуто без вопросов, вывести краткое резюме покрытия

## Особые соображения для написания романов

- **Жанровые конвенции**: Различные жанры (фэнтези, городское фэнтези, исторические) имеют разные акценты
- **Ожидания читателей**: Различные стандарты для коммерческих произведений и литературных произведений
- **Культурная чувствительность**: Некоторые темы требуют особой осторожности
- **Планирование серии**: Является ли это серией произведений, влияет на многие решения
