## Заметки по созданию

### Основные требования структуры «Семь точек»
- [ ] Каждая точка должна продвигать историю
- [ ] Переходы между точками должны быть естественными
- [ ] Поддерживать постоянное нарастание напряжения
- [ ] Середина должна действительно менять правила игры
- [ ] Точки сжатия нельзя пропускать
- [ ] Каждая точка должна повышать ставки

### Избегание частых проблем
- Слишком медленное начало (Hook)
- Недостаточная значимость первого сюжетного поворота (PP1)
- Слабые точки сжатия (Pinch points)
- Недостаточная сила середины (Midpoint)
- Недостаточная подготовка ко второму сюжетному повороту (PP2)
- Небрежная концовка (Resolution)

### Техники контроля темпа
- Использование коротких глав для ускорения темпа
- Смена POV в точках для увеличения воздействия
- Использование временного давления для поддержания напряжения
- Своевременные передышки для предотвращения усталости
