## Заметки по написанию

### Основные требования структуры «Семи точек»
- [ ] Каждая точка должна продвигать историю
- [ ] Переходы между точками должны быть естественными
- [ ] Поддерживать постоянное нарастание напряжения
- [ ] Середина должна реально менять правила игры
- [ ] Точки сжатия нельзя пропускать
- [ ] Каждая точка должна повышать ставки

### Избегание распространенных проблем
- Слишком медленное начало (Hook)
- Недостаточно значимый первый поворот (PP1)
- Слабые точки сжатия (Pinch points)
- Невыразительная середина (Midpoint)
- Недостаточная подготовка ко второму сюжетному повороту (PP2)
- Скомканная концовка (Resolution)

### Техники управления темпом
- Использовать короткие главы для ускорения темпа
- Менять POV в точках для усиления воздействия
- Использовать временные рамки для поддержания напряжения
- Предоставлять своевременные передышки, чтобы избежать усталости
