# Система управления предзнаменованиями

Мастер предзнаменований в стиле Шичжанъюй, управляющий тысячами скрытых нитей, создающий неожиданные развязки.

## Использование

```
/shizhangyu-foreshadow <операция> [параметры]
```

## Типы операций

- `add` - добавить новое предзнаменование
- `list` - просмотреть список предзнаменований
- `track` - отслеживать статус предзнаменований
- `activate` - активировать предзнаменование
- `resolve` - завершить предзнаменование
- `analyze` - анализировать структуру предзнаменований
- `suggest` - получить предложения по предзнаменованиям
- `check` - проверить цепочку логики

## Типы предзнаменований

### Предзнаменования-предметы
- Письма, секретные послания, жетоны, карты
- Предметы, кажущиеся обычными, но имеющие решающее значение
- Судьба утерянных предметов
- Источник таинственного подарка

### Предзнаменования-диалоги
- Случайно раскрытая информация
- Пророчества и предсказания
- Каламбуры и намеки
- Шутки, содержащие правду

### Предзнаменования-события
- Случайные встречи, кажущиеся совпадением
- Мелкие, но необычные события
- События, происходящие на фоне
- Упущенные детали

### Предзнаменования-персонажи
- Таинственное происхождение
- Скрытая личность
- Невысказанные секреты
- Пропавшие ключевые персонажи

## Расширенные функции

### Дизайн цепочек предзнаменований
Создание каскадных реакций между несколькими предзнаменованиями:
- Основная цепочка предзнаменований (ключевая тайна)
- Побочные цепочки предзнаменований (дополнительная информация)
- Преднамеренные предзнаменования (создание интриги)
- Предзнаменования-перевертыши (разрушение представлений)

### Управление временной шкалой
Управление предзнаменованиями в хронологическом порядке:
- Немедленные предзнаменования (в течение 1-3 глав)
- Короткие предзнаменования (5-10 глав)
- Средние предзнаменования (15-30 глав)
- Длинные предзнаменования (проходят через всю книгу)

### Валидатор логики
Проверка логической обоснованности предзнаменований:
- Установлена ли причинно-следственная связь?
- Соответствует ли хронология?
- Является ли объем информации адекватным?
- Своевременно ли время завершения?

## Стандарты вывода

1. **Система нумерации**: Единая нумерация для удобства отслеживания.
2. **Маркировка статуса**: Ожидает заложения / Заложено / Активировано / Завершено.
3. **Уровень приоритета**: Высокий / Средний / Низкий.
4. **Указание связей**: Отметка связанных предзнаменований.
5. **Напоминание о завершении**: Рекомендации по оптимальному времени.

## Связанные команды

- `/shizhangyu-style` - активация стиля
- `/shizhangyu-write` - написание предзнаменований
- `/shizhangyu-plot` - предзнаменования и интриги
- `/shizhangyu-character` - предзнаменования персонажей
