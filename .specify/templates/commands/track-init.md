# Инициализация системы отслеживания

На основе созданного плана истории и структуры глав инициализируются все файлы данных отслеживания.

## Когда использовать

Выполните эту команду после завершения работы над `/story` и `/outline`, перед началом написания.

## Процесс инициализации

1. **Чтение базовых данных**
   - Чтение `stories/*/story.md` для получения настроек истории
   - Чтение `stories/*/outline.md` для получения плана глав
   - Чтение `.specify/config.json` для получения метода письма

2. **Инициализация файлов отслеживания**

   Создание или обновление `spec/tracking/plot-tracker.json`:
   - Извлечение основных и второстепенных сюжетных линий из плана
   - Отметка ключевых сюжетных узлов
   - Установка точек отслеживания для заделов

   Создание или обновление `spec/tracking/timeline.json`:
   - Установка временных точек на основе плана глав
   - Отметка важных временных событий

   Создание или обновление `spec/tracking/relationships.json`:
   - Извлечение начальных взаимоотношений из описаний персонажей
   - Установка группировки фракций

   Создание или обновление `spec/tracking/character-state.json`:
   - Инициализация состояния персонажей
   - Установка начального местоположения

3. **Генерация отчета отслеживания**
   Отображение результатов инициализации для подтверждения готовности системы отслеживания

## Интеллектуальная связь

- Автоматическая установка контрольных точек в соответствии с методом письма
- Путешествие героя: контрольные точки для 12 стадий
- Трехактная структура: точки поворота для трех актов
- Семиточечная структура: 7 ключевых узлов

После инициализации системы отслеживания эти данные будут автоматически обновляться в процессе дальнейшего написания.
